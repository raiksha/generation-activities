# Actividad de Investigación

## 1. ¿Qué es un Sistema Operativo?

Un sistema operativo es el intermediario entre la parte física del computador, las aplicaciones y nosotros los usuarios. Nos provee de una interfaz para comunicarnos, ya sea por gráficos o texto. Su función principal es administrar los recursos físicos (CPU, RAM, almacenamiento) para que múltiples programas puedan ejecutarse de forma eficiente y segura. Otras funciones importantes incluyen controlar los periféricos (como teclado, mouse, audio, pantalla) y gestionar archivos (crear, leer, actualizar, borrar y mover).

Sin un SO, el computador sería inoperable para la mayoría, ya que no habría un software base que gestione procesos críticos como la asignación de memoria o la comunicación con periféricos. No existirían controladores automáticos ni una interfaz (gráfica o de comandos) para interactuar con la máquina. Tendríamos que programar instrucciones directamente en el lenguaje del procesador para cada tarea básica.

Los SO no se limitan solamente a computadores. La mayoría de aparatos electrónicos poseen un SO, como los celulares, Smart TVs, electrodomésticos inteligentes o consolas de videojuegos.

## 2. ¿Qué es una Terminal?

Una terminal es una interfaz de texto que te permite interactuar directamente con el SO. Es un programa donde sólo se utiliza texto. Sirve para ejecutar comandos y ejecutar tareas que la interfaz gráfica no puede o hace de forma más lenta. Los desarrolladores la usan con mucha frecuencia, porque te da acceso a funciones más avanzadas de lo que te permite la interfaz gráfica del SO. Tienes mayor control sobre qué tarea quieres que haga exactamente el SO, aparte de una mayor velocidad de ejecución. También consume bajos recursos y puedes automatizar múltiples comandos.

## 3. Preguntas de reflexión

### 1) ¿Es obligatorio tener un sistema operativo para programar? ¿Por qué?

**R:** Sí es necesario. Algunos definen a la programación como el arte de escribir código, y el escribir código puedes hacerlo hasta a mano en una hoja de papel (en ese caso, no necesitarías un SO). Pero para ejecutar ese código sí es necesario un SO, ya sea con interfaz gráfica o de texto.

### 2) ¿Por qué creen que muchos desarrolladores usan terminal en lugar de solo interfaces gráficas?

**R:** Los desarrolladores la usan con mucha frecuencia, porque te da acceso a funciones más avanzadas de lo que te permite la interfaz gráfica del SO.

### 3) ¿Qué ventajas podría tener usar la terminal?

**R:** Tienes mayor control sobre qué tarea quieres que haga exactamente el SO, aparte de una mayor velocidad de ejecución. También consume bajos recursos y puedes automatizar múltiples comandos.

### 4) ¿En qué situaciones creen que un programador usaría la terminal?

**R:** Cuando necesite control completo de lo que se instalará, cuando se necesiten instalar dependencias o para la administración remota de servidores.

## 4. Sobre README

### 1) ¿Qué es un README?

**R:** Es un archivo tipo carta de presentación que contiene información sobre un proyecto.

### 2) ¿Por qué los proyectos de programación suelen tener un archivo README.md?

**R:** Porque contiene información relevante sobre un proyecto.

### 3) ¿Dónde suelen aparecer los README?

**R:** En la página principal de un proyecto. Es el primer archivo que se abre cuando entramos a un link de un repositorio.

### 4) ¿Qué tipo de información suele incluir un README?

Un README suele incluir la explicación sobre qué hace un proyecto, qué problema resuelve y para qué sirve, instrucciones de instalación, una guía de uso, documentación para colaborar y qué tipo de licencia tiene.

---

## 5. ¿Por qué todo esto es importante para un desarrollador?

Todo esto es importante para un desarrollador, ya que son cosas que utilizará en el día a día en todo momento. El SO es donde se moverá, escribirá código y ejecutará programas; la terminal la utilizará para tareas más específicas, como instalar dependencias o administrar un servidor; y los README utilizará como medio de comunicación entre desarrolladores.
